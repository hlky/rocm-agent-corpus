# Eval Reports

Reports and attempt archives live here. A report should preserve enough
metadata for another agent to understand exactly what was compiled, tested,
benchmarked, compared, and classified.

Initial status: manifest-only.

Future report directories should use:

```text
eval/reports/<task_id>/
  attempts/
  summaries/
  leaderboards/
```

Reports should include wins, matches, losses, neutral results, compile
failures, correctness failures, and negative examples.
