# PyTorch API Real-World Shape Matrix

> ROCm parity note: This file was adapted from `H:/cuda-agent-corpus` on 2026-05-28. CUDA-origin benchmark numbers, source paths, and library names are comparison context only; do not cite them as ROCm evidence. New ROCm measurements must carry AMD hardware, ROCm version, compiler flags, gfx target, and an explicit evidence label.


This matrix is the admission gate for PyTorch-public-API HIP/ROCm challenge tasks.
Every PyTorch public API contract used by
`docs/PYTORCH_BASELINE_CHALLENGE.md` or
`eval/tasks/pytorch_baseline_challenge_manifest.json` must have at least one
real-world problem-size row here.

The shape rows are evidence that the problem size occurs in local Diffusers or
Transformers audits. They are not timing evidence and they are not claims that a
custom kernel is faster.

## Admission Rule

Use these labels consistently:

| Label | Meaning |
| --- | --- |
| `challenge-admitted` | The API may be used in a challenge task because at least one sourced real shape is listed below. |
| `candidate-with-shape` | A real shape exists, but the API is not yet part of a challenge task or scoring manifest. |
| `catalog-only` | The API appears in the public API catalog, but no real shape row is available yet. Do not build a challenge task from it. |
| `blocked-until-sourced` | The API family is plausible, but the local report scan found no concrete tensor sizes. |

When a new challenge task is added, include `real_shape_matrix_refs` in the task
manifest and cite the row IDs below.

## Catalog-Only Or Blocked APIs

The public catalog is intentionally broader than the current challenge. These
APIs must not be used as scored PyTorch baseline tasks until a sourced real
shape row is added:

| API family | Status | Reason |
| --- | --- | --- |
| `torch.linalg.solve`, `inv`, `qr`, `svd`, `eig`, `eigh`, `lu_factor`, `lstsq`, `matrix_exp` | `blocked-until-sourced` | The local Diffusers/Transformers report scan did not find concrete solver or decomposition tensor sizes. `torch.linalg.cholesky` has a narrowed Helios candidate row above. |
| Optimizers and `_foreach_*` | `blocked-until-sourced` | No concrete optimizer-state tensor lists were found in the requested reports. Add training workload sizes before scoring. |
| `torch.special.*` except sourced `torch.special.logit` | `catalog-only` | Plausible for activations/probability math, but only `torch.special.logit` has a sourced model-real candidate row so far. |
| `torch.nested.*` | `catalog-only` | Ragged sequence workloads exist, but no direct nested-tensor public API contract is anchored. |
| `kthvalue`, `msort`, `randperm`, `bernoulli`, `poisson`, generic `unique_consecutive` | `catalog-only` | Some training-only or optional rows exist, but these exact APIs are not yet sourced as inference challenge contracts. Dynamic `bucketize`, `searchsorted`, and `unique` have candidate rows above. |
| Arbitrary `grid_sample`, `unfold`, `fold`, and pooling variants | `candidate-with-shape` only | Real rows exist for SuperPoint, VideoMOT, PatchTGFXixer, AudioFlamingo3, and Allegro. Promote the bounded model subgraph rather than claiming generic API coverage. |

## Task Author Checklist

Before adding or accepting a PyTorch-public-API challenge task:

1. Pick the public reference expression or module.
2. Add at least one `real_shape_matrix_refs` value from this file.
3. Copy the concrete dimensions into `visible_shapes`.
4. State the shape suite and source report path.
5. If the API is only `candidate-with-shape`, promote it explicitly and explain
   the narrowed contract.
6. If the API is `catalog-only` or `blocked-until-sourced`, stop and gather a
   real model size first.
